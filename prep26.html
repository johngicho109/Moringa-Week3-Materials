        Why does variable scope matter?
It matters because it determines the extent to which the variable will be accessible. If local, it can only be used within the scope it is in but if it is global, then it can be accessed everywhere in the code.
What happens when you don't use var to declare a variable? Why is that bad?
If we don't use var, variable declared will be global even if it's supposed to be a local variable. It's a bad idea because it can collide with other variables in other parts of the code.
var rabbitName = "Njoro";

      function hoppity() {
        alert(rabbitName);
        rabbitName="Fredi";
      }

      hoppity();
      alert(rabbitName);Under the above code.First alert will give "Njoro".This is because it's alerted before the update in the variable is done.The secodn alert will give "Fredi".This is because, the function is invoked before alerting, hence picking the update.
var rabbitName = "Njoro";

      function hoppity() {
        var rabbitName = "Tanu"
        alert(rabbitName);
        rabbitName="Fredi";
      }

      hoppity();
      alert(rabbitName);
Under the above code.First alert is "Tanu". This is because, the function is invoked and the alert is immediately after the update(from Njoro to Tanu) on the variable is done.Second alert gives "Njoro". This is because the all the alerts on the local variables has been done once the function was invoke.Hence the alert after function inking pics the global name Njoro.
Why do you need event.preventDefault() when working with jQuery forms?
event.preventDefault() allows the html/page to load fully first before it's acted upon by JQuery. We also need it when we want to display user inputs from a form. This prevent the page from reloading and making us loose user input from the screen.
